About me

Hello! My name is Milan Dewilde and I’m a student of Applied Computer Science at University College Ghent.

I like climbing, hiking, surfing and travelling in general. I am a very social and optimistic person with a passion for technology. I like working in teams, taking responsibility for my work and making sure that I always do my part. I also enjoy trying to understand complex systems and I always try to keep up with innovations. To me, contributing to the innovation and replacement of obsolete technology is a process I'd love to be a part of and I hope to be able to make a job out of this passion one day.

Recently I discovered that computer networks and systems management is the field that interests me most, and so I've decided to look into some online courses on cloud storage and services to keep myself in the loop on the latest technologies.

I’ve created this website to serve as a professional diary in which I keep track of my achievements and things I’ve learned along the way.

Pazaak
A Java based card game

My first real programming project was Pazaak, a card game featured in the Star Wars universe. The objective was to create the game, as well as to create a database and to configure integration for logging in users, registering new users and saving games and high scores. This project was also the first time I worked in a team to complete a large task, which was a very good learning experience, and also the first time we used the Scrum-principles.

The application itself was programmed in Java. Database software was MySQL and to create the GUI, we used JavaFX.






Data in display
Processing and visualizing data using PowerBI and SQL

The goal for this project was to create queries to extract data from a database. After we had extracted this data, we had to select relevant data to create a datawarehouse. Lastly, using the queries we had created and applying them to our newly created datawarehouse, we had to create graphs using Microsoft's PowerBI.

This project gave me a much better understanding of SQL, as well as further enhancing my skills for working in teams. Creating graphs from data we had extracted in very specific ways was a very interesting exercise with a lot of potential for real-world application.





Cybersecurity
An online course by Cisco

An introduction into the world of Cybersecurity, provided by Cisco. I took this course because network security is an interesting topic to me, and I felt that no other course in school properly prepared us for possible dangers to network infrastructure. The course taught me the very basics of Cybersecurity, as well as clarifying the most important and dangerous forms of cyberattacks

in this course we used Cisco's Packet Tracer to examine some network attacks and vulnerabilities, and Wireshark to exemplify the vulnerabilites of certain transfer protocols.

Network topologies
Exploring network possibilities

Creating network topologies is something I really enjoy doing. It's very interesting to me to see and understand how things are connected. I hope to further expand my knowledge on this subject in the future.

Topologies I made were usually simulations in Packet Tracer. However, I made some simple topologies on physical routers and switches as well. All of this was done on Cisco devices.





Webserver
A LAMP-stack with automated and scripted installation

Creating this webserver was by far my favourite project so far. The objective was to automate the deployment of a Virtual Machine that was running a LAMP-stack. For this project I had to learn scripting, as well as Linux and PHP. It was the most difficult project I've had so far, but definately the most rewarding upon completion. I wrote scripts that automated the installation of all required software, as well as creating a database and scripts for backing up said database to a different virtual server. After deployment was complete, I wrote a very simple web-application for loging in members that were registered within the database and displaying content based on their permissions.

The installation script were mostly shell scripts and were run by Vagrant. The web-application was written in PHP, HTML and CSS.





Configuring network devices

A Cisco course teaching initial network device configuration and setting up routes, as well as more complex routing protocols and inter-VLAN networking. This course was very interesting to me because I enjoy understanding how things are connected.

Most configurations were done in Packet Tracer, but I also did some on physical Cisco devices.
